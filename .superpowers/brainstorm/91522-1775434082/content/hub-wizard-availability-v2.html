How Term-Type Availability Works

  • Three tabs: Fall / Spring / Summer — each has its own availability grid
  • Pre-population: When a new term is created (e.g., Fall 2027), the instructor's Fall availability template auto-populates. Editable per-term afterward.
  • "Copy Fall → Spring" quick action — since Fall and Spring are often similar
  • Status cards at bottom — at-a-glance view of which term types have been configured
  • Not set = all available — unset term types default to fully available (no restrictions)
  • Also applies to the Availability tab in the hub — same Fall/Spring/Summer sub-tabs when editing availability on the detail view
