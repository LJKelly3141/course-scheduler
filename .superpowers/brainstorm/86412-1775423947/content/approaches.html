Instructor UI Redesign — Three Approaches

Click the approach that feels right for your workflow

A

Instructor Hub (Recommended)

One unified page with smart sections

Replace the scattered list page + detail page with a single Instructor Hub. Left panel shows the instructor roster; right panel shows the selected instructor's full profile, availability, notes, and workload — all in one view without tab-switching.

  • Master-detail layout (list ↔ detail side by side)
  • Guided "Add Instructor" wizard for onboarding
  • Inline load report preview with "Export for Dean" button
  • Term selector filters availability + workload contextually

Pros

  • Everything in one place — no navigation
  • Natural flow: select → review → edit → export
  • Scales well as instructor count grows

Cons

  • More complex to build
  • Needs responsive handling for smaller screens
B

Polished Current Pattern

Keep list + detail pages, but streamline and connect them

Keep the two-page structure but fix the inconsistencies. Better list page with clear actions, improved detail page with logical sections, and a dedicated Load Report page/tab.

  • Clean up the list page (remove embedded availability)
  • Reorganize detail page tabs for logical flow
  • Add "New Instructor" guided form (modal or page)
  • Separate Load Report view with preview + export

Pros

  • Less disruptive — familiar patterns
  • Simpler implementation
  • Works well on any screen size

Cons

  • Still requires navigating between pages
  • Doesn't fully solve the "scattered" feeling
C

Workflow-Oriented Sections

Organize by task rather than by entity

Instead of an "Instructors" page, create workflow-specific views: an Onboarding flow, a Roster Management view, a Term Assignment view, and a Load Reports view — each optimized for its specific task.

  • Each view shows only what's relevant to that task
  • Onboarding wizard walks through profile → availability → done
  • Assignment view shows instructors alongside schedule grid
  • Load report view with live preview and export

Pros

  • Each workflow is focused and clean
  • Reduces cognitive load per task
  • Assignment view could integrate with scheduling

Cons

  • Biggest redesign — most work
  • Instructor data split across multiple views
  • "Where do I go to change X?" problem
