# Tips & Shortcuts

Helpful shortcuts, conventions, and workflow tips to get the most out of Course Scheduler.

## Keyboard Shortcuts

| Shortcut | Action |
|----------|--------|
| **Ctrl+Z / Cmd+Z** | Undo last action |
| **Ctrl+Y / Cmd+Y** | Redo last undone action |

Undo and Redo are also available as buttons in the top header bar.

## Day Codes

The app uses these standard day codes throughout:

| Code | Day |
|------|-----|
| **M** | Monday |
| **T** | Tuesday |
| **W** | Wednesday |
| **Th** | Thursday |
| **F** | Friday |
| **S** | Saturday |
| **U** | Sunday |

Common patterns: **MWF** (Mon/Wed/Fri), **TTh** (Tue/Thu), **MW** (Mon/Wed)

> **Important:** Thursday is **Th**, not "R". This differs from some campus systems that use "R" for Thursday.

## Standard Time Blocks

The scheduling grid uses these standard time block patterns:

| Pattern | Duration | Range |
|---------|----------|-------|
| **MWF** | 50 minutes | 8:00 AM – 4:50 PM |
| **TTh** | 75 minutes | 8:00 AM – 4:45 PM |
| **Evening** | 170 minutes | 6:00 PM start, Mon–Thu |

You can also create **custom time blocks** when scheduling a meeting if the standard blocks don't fit.

## Theme

Click the **theme toggle** in the sidebar footer to cycle between:
- **System** — Follows your OS dark/light setting
- **Light** — Always light mode
- **Dark** — Always dark mode

## Workflow Best Practices

### Start of Semester Planning

1. **Copy the previous term** — Use the Copy Term feature on the Terms page to start from last year's schedule
2. **Import updates** — Upload the latest room, instructor, and course changes
3. **Review the dashboard** — Check for missing courses and instructor changes
4. **Adjust and resolve** — Use the Schedule Grid to make changes and resolve conflicts
5. **Finalize and publish** — Lock the term and export/push to GitHub Pages

### During the Semester

- Use **instructor notes** to record preferences, constraints, and feedback for next term
- Track **enrollment history** by importing enrollment data after census dates
- Review **analytics** to identify courses that need more or fewer sections

### Summer Scheduling

- Create sessions (A, B, Full, etc.) with specific date ranges
- Assign sections to sessions for proper date tracking
- Use the **Paste Sessions** feature to quickly import session tables from registrar documents

## Auto-Save

Everything saves automatically. There is no manual save button. When you:
- Edit a field and click away → saved
- Drag and drop a meeting → saved
- Toggle availability → saved
- Dismiss a warning → saved

## Database Safety

Your schedule data is stored in a local SQLite database. To protect your data:
- **Download backups** regularly from Settings
- The database file is shown in Settings so you can find it for manual backup
- App updates preserve all your existing data — no data is ever overwritten during updates
